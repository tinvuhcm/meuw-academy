# Audit Tổng Thể Ứng Dụng - 2026-06-15

## Phạm vi

Audit này gom 5 mặt:

- nội dung và curriculum
- chức năng runtime
- bảo mật và dữ liệu
- hiệu năng và độ mượt
- rác kỹ thuật / độ sạch repo / khả năng vận hành trên Vercel + Supabase

Bối cảnh hiện tại:

- app là static SPA trên Vercel
- tài khoản và cloud backup dựa trên Supabase
- mục tiêu người dùng nêu ra là chuẩn hóa app để mượt, ít bug, đủ an toàn, và có thể phục vụ khoảng 100 users cùng dùng trên hạ tầng hiện tại

## Tóm tắt điều hành

Kết luận ngắn:

- Về hạ tầng, Vercel + Supabase ở mô hình hiện tại đủ sức cho cỡ 100 users nếu traffic chỉ ở mức app học tập nhỏ đến vừa.
- Về chất lượng ứng dụng, codebase hiện chưa ở trạng thái có thể tự tin gọi là "đủ sạch và đủ chắc" để mở rộng cộng đồng ngay bây giờ.
- Có ít nhất 1 bug chức năng chắc chắn, 1 log debug lọt vào runtime, 1 script audit chưa sạch môi trường Node, và nhiều artefact rác ở root repo làm tăng rủi ro vận hành / deploy / bảo trì.
- Có một số rủi ro bảo mật mức vừa do chính sách parent PIN mặc định và do chưa chứng minh được lớp bảo vệ backend thực tế ngoài tài liệu Supabase.

## Findings

### [Cao] Bug đổi tên hồ sơ đang gán sai biến và có thể lỗi runtime

- File: `js/state.js:347-349`
- Bằng chứng:

```js
function updateProfileName(id, newName) {
  getActiveProfile().name = name;
  commit();
}
```

Đánh giá:

- Hàm nhận `newName` nhưng lại gán từ biến `name` không tồn tại trong scope.
- Đây là bug chức năng thật, có thể gây `ReferenceError` hoặc khiến tính năng đổi tên hồ sơ không hoạt động đúng.

Tác động:

- ảnh hưởng trực tiếp tới chức năng profile
- làm giảm độ tin cậy của bề mặt quản lý hồ sơ

Khuyến nghị:

- sửa ngay trong pha fix đầu tiên
- thêm kiểm tra hẹp cho bề mặt profile rename

### [Cao] Log debug lọt vào runtime curriculum và làm bẩn console / script output

- File: `js/data/fresh-curriculum.js:1253`
- Bằng chứng:

```js
if (index === 11) console.log('ORDER 11:', subjectOrder);
```

Đánh giá:

Tác động:

Khuyến nghị:

- bỏ log này ngay
- rà thêm console log không cần thiết trong runtime app

### [Cao] Script audit nội dung chưa headless-safe vì phụ thuộc `localStorage` nhưng không tự stub

Khuyến nghị:
